## Objetivo

Definir e priorizar 200+ keywords com foco em intenção de compra para presentes geek, cobrindo head, mid e long-tail, segmentadas por persona, ocasião e faixa de preço.

---

## Campos do Backlog (Planilha)

| Campo | Descrição |
|---|---|
| `keyword` | Termo principal |
| `volume` | Busca mensal (Brasil) |
| `kd` | Dificuldade (0–100) |
| `intent` | Informacional / Comercial / Transacional |
| `persona` | Ana / Lucas / Marina |
| `cluster` | Ocasião / Perfil / Preço / Categoria |
| `funnel_stage` | Awareness / Consideration / Decision |
| `target_url` | URL (existente ou a criar) |
| `content_type` | product_single / listicle / guide / hub |
| `priority` | Alta / Média / Baixa |
| `status` | Ideia / Brief / Em produção / Publicado / Atualizar |
| `notes` | Observações (concorrentes, ângulo de copy) |

---

## Priorização

- Quick wins: KD baixo e alta intenção ("presentes geek até 100 reais").  
- Sazonais: priorizar até 90 dias antes (Natal, BF, Namorados).  
- Hubs: criar primeiro as pillar pages e depois os clusters.

---

## Concorrência

- Levantar top 5 concorrentes orgânicos por cluster (ex.: Natal, Gamer, Dev).  
- Analisar lacunas: tópicos não cobertos, conteúdos ultrapassados, ausência de FAQ/tabelas.  
- Planejar superação: conteúdo mais completo, melhor UX, snippets (FAQ/HowTo/listas).
